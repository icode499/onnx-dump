# Basic Add Relu Example Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the `/tmp`-based quick-start example with a self-contained `examples/basic_add_relu/` directory that includes reproducible input and output artifacts.

**Architecture:** Move the example into a named directory with `input/` and `output/` subdirectories, expose a reusable `generate_example()` function for testability, and regenerate the checked-in example artifacts from the script itself.

**Tech Stack:** Python, pathlib, shutil, numpy, onnx, pytest

**Spec:** `docs/superpowers/specs/2026-03-26-basic-add-relu-example-design.md`

---

## File Structure

| File | Responsibility |
|------|---------------|
| `examples/basic_add_relu/generate.py` | Build the example model and generate deterministic input/output artifacts |
| `examples/basic_add_relu/input/model.onnx` | Example ONNX model checked into the repo |
| `examples/basic_add_relu/input/tensors/X.npy` | Input tensor for graph input `X` |
| `examples/basic_add_relu/input/tensors/Y.npy` | Input tensor for graph input `Y` |
| `examples/basic_add_relu/output/manifest.json` | Example manifest generated by `dump_model()` |
| `examples/basic_add_relu/output/tensors/*.npy` | Example dumped tensors |
| `tests/test_examples.py` | Regression tests for the example generator layout and contents |
| `README.md` | User-facing example instructions |
| `examples/run_example.py` | Removed legacy entry point to avoid duplicate example layouts |
